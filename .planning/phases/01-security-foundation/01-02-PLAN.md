Add these two functions. Place them right after the `writeSiteData` function (currently around line 219 after Plan 01's atomic write changes):
```javascript
function createBackup() {
    if (!fs.existsSync(BACKUP_DIR)) fs.mkdirSync(BACKUP_DIR, { recursive: true });
    var ts = new Date().toISOString().replace(/[:.]/g, '-');
    var files = [DATA_FILE, SITE_DATA_FILE];
    files.forEach(function (f) {
        if (fs.existsSync(f)) {
            fs.copyFileSync(f, path.join(BACKUP_DIR, path.basename(f, '.json') + '-' + ts + '.json'));
        }
    });
    cleanOldBackups();
}

function cleanOldBackups() {
    if (!fs.existsSync(BACKUP_DIR)) return;
    var files = fs.readdirSync(BACKUP_DIR).filter(function (f) { return f.endsWith('.json'); });
    files.sort();
    while (files.length > MAX_BACKUPS) {
        fs.unlinkSync(path.join(BACKUP_DIR, files.shift()));
    }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Untrusted input — rate limiting prevents abuse |
| Server → HTTP Response | Security headers protect client-side interactions |
| Server → Filesystem | Backups protect against data loss |
| Server → npm Packages | Third-party code (helmet, compression, express-rate-limit) trusted as industry-standard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Spoofing | All pages | mitigate | SEC-06: helmet sets X-Frame-Options: SAMEORIGIN — prevents clickjacking via iframe embedding |
| T-01-05 | Tampering | All responses | mitigate | SEC-06: helmet sets X-Content-Type-Options: nosniff — prevents MIME type sniffing attacks |
| T-01-06 | Information Disclosure | HTTP headers | mitigate | SEC-06: helmet removes X-Powered-By: Express header — prevents server fingerprinting |
| T-01-07 | Information Disclosure | HTTPS | mitigate | SEC-06: helmet sets Strict-Transport-Security header — enforces HTTPS |
| T-01-08 | Denial of Service | /api/* endpoints | mitigate | SEC-08: apiLimiter 200 req/15min prevents API abuse and scraping |
| T-01-09 | Denial of Service | /api/login | mitigate | SEC-08: loginLimiter 5 req/15min prevents brute-force password attacks |
| T-01-10 | Denial of Service | Data loss | mitigate | SEC-04: Automatic backups every 6 hours with rotation of 10 copies |
| T-01-11 | Denial of Service | Bandwidth | accept | SEC-07: compression reduces bandwidth ~60% but does not prevent DDoS — acceptable for single-server deployment |
</threat_model>
