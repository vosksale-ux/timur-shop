# Requirements: Timur.Shop

**Defined:** 2026-04-06
**Core Value:** Покупатель может легко выбрать и заказать кожаное изделие, а владелец — управлять каталогом и заказами через удобную админку

## v1 Requirements

### Security

- [ ] **SEC-01**: Пароль администратора загружается только из env var ADMIN_PASSWORD, без fallback (сервер не стартует без него)
- [ ] **SEC-02**: SVG убран из разрешённых форматов загрузки изображений (предотвращает stored XSS)
- [ ] **SEC-03**: Все JSON-файлы записываются атомарно (write → temp file → rename), предотвращая повреждение при сбое
- [ ] **SEC-04**: Автоматические бэкапы data/ файлов с вращением (хранится N последних копий)
- [ ] **SEC-05**: Реальные контактные данные заменяют фейковые (+7999000000, href="#")
- [ ] **SEC-06**: Security headers через helmet (CSP без strict пока, X-Content-Type-Options, X-Frame-Options и т.д.)
- [ ] **SEC-07**: Gzip сжатие ответов через compression middleware
- [ ] **SEC-08**: Rate limiting на всех API эндпоинтах через express-rate-limit

### Orders

- [ ] **ORD-01**: Покупатель может оформить заказ через форму на странице корзины, заказ сохраняется в data/orders.json
- [ ] **ORD-02**: Заказ содержит: имя, телефон, адрес/комментарий, способ доставки, товары, сумма, дата, статус
- [ ] **ORD-03**: Заказы имеют статусы: Новый → Подтверждён → В работе → Отправлен → Выполнен → Отменён
- [ ] **ORD-04**: Владелец получает уведомление о новом заказе в Telegram через Bot API
- [ ] **ORD-05**: Владелец может просматривать список заказов в админке (admin-orders.html)
- [ ] **ORD-06**: Владелец может менять статус заказа в админке
- [ ] **ORD-07**: Владелец может просматривать детали заказа (товары, контакты покупателя)
- [ ] **ORD-08**: Корзина очищается только после успешного сохранения заказа на сервере

### Reviews

- [ ] **REV-01**: Покупатель может оставить отзыв с текстом, именем и оценкой (1-5 звёзд)
- [ ] **REV-02**: Все новые отзывы сохраняются в data/reviews.json со статусом pending
- [ ] **REV-03**: Владелец может модерировать отзывы (одобрять/удалять) в админке
- [ ] **REV-04**: Одобренные отзывы отображаются на странице reviews.html динамически через API
- [ ] **REV-05**: Существующие 8 статических отзывов мигрированы в reviews.json как approved
- [ ] **REV-06**: На странице отображается средний рейтинг и количество отзывов

### UI & Polish

- [ ] **UI-01**: Мобильная адаптивность — исправить поехавшие секции, корректное отображение на всех экранах
- [ ] **UI-02**: Плавные CSS-анимации и hover-эффекты на карточках товаров, кнопках, навигации
- [ ] **UI-03**: Компоненты интерфейса выглядят аккуратно (отступы, выравнивание, размеры)

## v2 Requirements

### Payments

- **PAY-01**: Онлайн-оплата картой через платёжный шлюз (ЮKassa или Тинькофф)

### Enhancements

- **ENH-01**: Поиск по каталогу товаров
- **ENH-02**: Сортировка товаров (по цене, имени)
- **ENH-03**: Пагинация каталога при росте числа товаров
- **ENH-04**: Отслеживание заказа покупателем по номеру
- **ENH-05**: Фото-отзывы (покупатель прикрепляет фото к отзыву)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Регистрация покупателей | Заказ без аккаунта — проще для конверсии, масштаб не требует |
| Онлайн-оплата (v1) | Сначала стабильная система заказов, потом платёжный шлюз |
| Полный редизайн | Текущий дизайн устраивает, нужны фиксы |
| Переход на БД | JSON-хранилище достаточно при атомарной записи и бэкапах |
| CMS/Блог | Не нужен для магазина из 18 товаров |
| Рекомендации/алгоритмы | Слишком малый каталог для ML |
| Мобильное приложение | Web-first, мобильный сайт достаточен |
| Email-уведомления | Telegram покрывает потребность владельца |
